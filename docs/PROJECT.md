# A.M. Target — Лендінг: інфраструктура проекту

Окремий статичний лендінг для **amtarget.net**, що візуально 1:1 успадковує сайт
(шапка/футер/компоненти), а нові секції будує під `.landing` за дизайн-системою.

## Бриф

- **Тема:** мішені для **бойової підготовки ЗСУ** (тактична/військова аудиторія).
- **Бренд і дизайн-система:** ті самі, що в магазину amtarget.net (див. `DESIGN-SYSTEM.md`).
- **Джерело контенту:** умовно ч/б **Figma-прототип** → беремо **лише текст і структуру секцій**;
  стилі прототипу **не використовуємо** (сирі/неточні) — оформлення лише за нашою дизайн-системою.

## Структура

```
A.M.Target/
├─ docs/                          # 📚 база знань проекту (пайплайн-артефакти)
│  ├─ PROJECT.md                  # цей файл — інфраструктура й етапи
│  ├─ DESIGN-SYSTEM.md            # дизайн-система, витягнута з сайту  ← ЕТАП 1
│  ├─ design-tokens.css           # токени як CSS-змінні (для верстки)
│  ├─ SECTIONS.md                 # план секцій лендінгу (контент+структура) ← ЕТАП 2
│  └─ reference/                  # скріншоти/референси з живого сайту
│
└─ landing frame/                 # 🏗️ ціль збірки (готовий статичний пакет)
   ├─ index.html                  # розмітка (шапка + <main class="landing"> + футер)
   ├─ style.css                   # стилі лендінгу (нові правила під .landing)
   ├─ README.md                   # правила роботи з каркасом
   └─ assets/{css,fonts,img}      # повний CSS теми + шрифти + графіка
```

## Пайплайн (етапи роботи)

| # | Етап | Вхід | Артефакт | Статус |
|---|---|---|---|---|
| **1** | **Дизайн-система** — витягти токени й компоненти з сайту | amtarget.net + theme CSS | `DESIGN-SYSTEM.md`, `design-tokens.css` | ✅ готово |
| **2** | **Структура секцій** — витягти текст+структуру з Figma-прототипу, скласти план секцій | Figma + DS | `SECTIONS.md` | ⬜ наступний |
| **3** | **Макет (дизайн)** — перекласти прототип на нашу дизайн-систему | SECTIONS + DS | in-code прев'ю / Figma | ⬜ |
| **4** | **Верстка** — реалізувати секції в `<main class="landing">` | макет + DS | `index.html` + `style.css` | ⬜ |
| **5** | **QA** — брейкпоінти, іконки, шрифти, без глобальних оверрайдів | готова сторінка | чеклист (README §9) | ⬜ |

## Принципи

- **Перевикористання > переписування.** Шапка, футер, кнопки, `.richtext`, `shortcard`
  вже стилізовані темою — беремо як є.
- **Ізоляція нового.** Усі нові правила — під `.landing` (BEM `.landing__*`). Жодних
  глобальних оверрайдів і жодного перевизначення класів сайту для нових блоків.
- **Mobile-first.** Брейкпоінти 359 / 768 / 1024 / 1200 / 1400 / 1800.
- **Єдине джерело істини стилів сайту** — `landing frame/assets/css/styles.css`
  (знімок теми; оновлюється перекопіюванням).

## Локальний запуск

```bash
npx serve "landing frame"     # або VS Code Live Server (інакше file:// блокує SVG-спрайт)
```
